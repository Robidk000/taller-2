lecciones aprendidas:
1. conexion entre python y la base de datos
   configurar sqlalchemy requiere atencion a sus conexiones.
   separar la logica de conexion en un archivo independiente mejora la organizacion.

2. diseño de la base de datos
   es importante definir correctamente las entidades y sus relaciones antes de programar.
   tener el esquema bien normalizado facilita la integracion con el codigo y evita redundancias.

3. gestion del repositorio y reproducibilidad
   mantener una estructura solida de carpetas ayuda a localizar rapidamente el codigo y los recursos.

   4. creacion de la api rest con flask
      es importante validar los datos que recibe el endpoint para evitar errores y mantener la seguridad.

   la modularidad de flask permite organizar rutas en bluepoints, lo que hace el proyecto mas escalable.

5. reflexion personal
   el proceso mostró que un proyecto backend no solo depende de escribir el codigo, sino tambien de pensar y probrar.
